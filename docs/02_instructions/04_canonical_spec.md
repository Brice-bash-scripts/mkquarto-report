# mkquarto_report Canonical Specification

This document consolidates the final, authoritative requirements for the mkquarto_report scaffold script.

## 1. Project Naming

- The script accepts one required positional argument: project name.
- The target folder name must use the slugified title.
- Example:
  - Input: Real-Estate-Prediction
  - Folder: real-estate-prediction
  - Human title: Real Estate Prediction

## 2. Folder Structure

The generated project must follow this structure:

```text
<slugified-project-name>/
├── report.qmd
├── _quarto.yml
├── references.bib
├── README.md
├── .mkquarto/
│   └── generated_files_manifest.txt
├── content/
│   ├── 00_frontmatter/
│   │   ├── 00_0_cover.qmd
│   │   └── 00_1_structure_overview.qmd
│   ├── section_01/
│   ├── section_02/
│   ├── section_03/
│   ├── section_04/
│   ├── section_05/
│   ├── section_06/
│   ├── section_07/
│   ├── section_08/
│   ├── section_09/
│   ├── section_10/
│   ├── figures/
│   │   └── figures.qmd
│   └── appendices/
│       └── appendices.qmd
├── assets/
│   ├── images/
│   └── tables/
└── output/
```

## 3. Template Sources And Output Extensions

- Template instruction files remain in markdown under docs/02_instructions/templates.
- The script must generate Quarto files with .qmd extension.
- Canonical naming authority is the template filename conventions.
- Mismatched names in older examples are not authoritative.

## 4. Title Substitution

- Placeholder token: {{title}}
- The script may also support legacy ${title} replacement for compatibility.
- Human-readable title should be substituted in generated content where placeholders exist.
- Slug should be used for project directory naming.

## 5. Canonical report.qmd Include Order

The include order must be:

1. content/00_frontmatter/00_0_cover.qmd
2. content/00_frontmatter/00_1_structure_overview.qmd
3. All section_01 through section_09 files in numeric order
4. content/figures/figures.qmd
5. content/appendices/appendices.qmd
6. content/section_10/10_0_references.qmd

## 6. Table Of Contents Strategy

- TOC is controlled only in _quarto.yml.
- Do not hard-code TOC generation in report.qmd.

## 7. References Section Mechanics

- content/section_10/10_0_references.qmd must be generated from the section_10 template source.
- Do not hardcode a special minimal references section in the script.

## 8. Idempotency And Force Behavior

Default rerun behavior:

- Existing files are skipped.
- Skip actions should emit clear warning messages.

Force mode:

- A --force flag is required.
- --force may overwrite only files tracked as generated by the manifest.
- User-created or untracked files must not be overwritten.

## 9. Manifest Requirements

- Manifest path: .mkquarto/generated_files_manifest.txt
- Manifest must be updated on every run.
- Manifest must be pruned on every run so it only lists currently generated paths.

## 10. Validation Modes

Validation behavior:

- Default mode: full render validation.
- Fast mode: quarto check only.
- Full mode failure must return a non-zero exit code.
- If full validation fails, scaffolding should still be created and retained.

## 11. Output Directory

- Quarto output directory must be output/.

## 12. README Scope

- The script should scaffold a minimal README.
- Users can expand README content after generation.

## 13. CLI Requirements

The script must support:

- --help
- --force
- validation mode selection (default full, fast mode available)

The script must run non-interactively and be suitable for automation.
