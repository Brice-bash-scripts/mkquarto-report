## What This Project Does

The script generates a complete Quarto report scaffold with:

- canonical directory structure
- preloaded section templates converted to qmd
- report entrypoint and Quarto configuration
- bibliography file and starter README in generated project
- idempotent reruns with safe overwrite control via manifest

## Quick Start

Run from the directory where you want the new report folder created.

```bash
chmod +x src/mkquarto_report
./src/mkquarto_report Real-Estate-Prediction
```

Example output folder:

```text
real-estate-prediction/
```

## CLI Usage

```text
mkquarto_report [options] <project-name>
```

Options:

- -f, --force
  - overwrite only files previously generated by this script
- --validation <mode>
  - validation mode: full | fast | none
- --fast-validate
  - shortcut for --validation fast
- --no-validate
  - shortcut for --validation none
- -h, --help
  - show usage help

Examples:

```bash
./src/mkquarto_report Real-Estate-Prediction
./src/mkquarto_report --fast-validate Mortgage-Analytics
./src/mkquarto_report --force --validation full Customer-Churn-Study
```

## Validation Modes

- full (default)
  - runs: quarto render report.qmd --to pdf
- fast
  - runs: quarto check
- none
  - skips validation

If full validation fails, scaffold creation is preserved and the script exits non-zero.

## Idempotency And Safety Model

- Existing files are skipped by default with warning logs.
- Force mode only overwrites files listed in:

```text
.mkquarto/generated_files_manifest.txt
```

- Untracked or user-created files are not overwritten by force mode.

## Exit Codes

- 0: scaffold succeeded and validation passed or was skipped
- 1: scaffold created but validation/dependency check failed
- 2: invalid CLI usage or argument values
